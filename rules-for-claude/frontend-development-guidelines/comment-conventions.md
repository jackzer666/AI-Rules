# Comment Conventions

- Place comments on their own line above the code. No inline/end-of-line comments.
- Use `//` by default. Use `/* */` or `<!-- -->` where `//` is invalid.
- Add one space after `//` (e.g., `// Comment`).
- For multi-line blocks, place comment immediately above with no blank line.
- `/** */` allowed for API docs and complex exported functions.
- No emoji. Write clear, human-sounding English.
